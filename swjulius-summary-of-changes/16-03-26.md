## Navigation / tab performance

### `app/(tabs)/_layout.tsx`
- Updated `Tabs` `screenOptions` to reduce tab-switch lag.

## History screen (backend-derived history)

### `app/history.tsx`
- History screen no longer reads from AsyncStorage cache.
- History now reads from a **time-series** table intended for snapshots:
  - `energy_readings_log` with a timestamp column `snapshot_at`.
- UI shows only **available** groups based on existing data:
  - Horizontal filter chips for `Today`, `Yesterday`, `This Week`, `This Month`, etc. (only if data exists).
- Events are derived by diffing `detected_appliances` between consecutive snapshots:
  - appliance appears -> "turned on"
  - appliance disappears -> "turned off"

## Backend plan (Supabase) – required for real History

To make History work with real groups, the backend needs to **store snapshots over time** (because `energy_readings` currently looks like a single latest row).

Recommended backend additions (additive; does not overwrite existing tables):
- New table: `energy_readings_log`
- Scheduled snapshot job: every **30 minutes**
- Retention cleanup: delete rows older than **1 month**

(See chat for the SQL suggested to create the table, snapshot function, cleanup function, and `pg_cron` schedules.)
